ThreadScout

ThreadScout is a local-first automation platform for clothing resellers. It streamlines the transition from raw product photography to complete, market-ready eBay drafts using AI Vision.

What ThreadScout Provides

  • AI Vision Extraction: Automatically identifies brand, size, material, and condition from garment photos using Gemini 2.0 Flash.
  • Market Research & Enrichment: integrated Google Lens and eBay Sold Comps with “Paste & Scrape” enrichment to pull catalog data and pricing into your listings.
  • One-Click eBay Drafts: Pushes finalized metadata and images directly to your eBay account as a structured draft.
  • Local-First Storage: Organizes your inventory photos into a human-readable, slugified hierarchy on your own hardware.
  • Hardware Isolation: Optional QEMU MicroVM support for running AI tasks in a secure, restricted sandbox.

Quick Start: Deployment

Get the app running locally for personal use in less than 5 minutes.

  1. Clone & Enter: git clone https://github.com/mikeco/threadscout.git && cd threadscout
  2. Install Mise: curl https://mise.jdx.dev/install.sh | sh
  3. Trust Configuration: mise trust (Approves the orchestration tasks)
  4. Setup Environment: mise run setup (Installs CA certs and browsers)
  5. Launch: mise run dev
  6. Access: https://localhost:3443

For detailed installation and Cloudflare OAuth configuration, see DEPLOYMENT.

Quick Start: Developer Onboarding

For those looking to contribute or customize the internals.

  1. Clone & Enter: git clone https://github.com/mikeco/threadscout.git && cd threadscout
  2. Trust & Setup: mise trust && mise run setup
  3. Install Dependencies: mise run install
  4. Secrets: Initialize your fnox store and add your EBAY_CLIENT_ID.
  5. Mock Mode: Run APP_ENV=mocking THREADSCOUT_ENABLE_AUTH=true mise run dev to test UI and auth without hitting real APIs.
  6. Learn the Internals: Read ARCHITECTURE for the high-level maps and philosophy.

Frequently Asked Questions (FAQ)

Q: Where is my data stored? A: All item metadata is stored in a local SQLite database (backend/data/threadscout.db), and images are stored in the storage/ directory on your machine.

Q: Does ThreadScout share my photos with eBay immediately? A: No. ThreadScout only pushes data to eBay when you explicitly click “Push to eBay.” Everything stays local until you are ready to list.

Q: What AI model does it use? A: We use Gemini 2.0 Flash for visual analysis and Gemini 2.5 Flash-Lite for high-speed text enrichment. Both run on Google Vertex AI.

Q: Can I use this for things other than clothes? A: While the current prompts are optimized for garments, the architecture is designed to support different “Category Profiles” in the future.
