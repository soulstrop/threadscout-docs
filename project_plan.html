Phase 4: eBay API Integration (Completed)

    • Secure storage for App ID, Dev ID, and Cert ID.
    • Database schema update for OAuth2 Refresh Tokens.
    • At-Rest Encryption: Fernet encryption for tokens using fnox-managed keys.
    • CSRF Protection: Added cryptographically securestate parameter verification.
    • Stable Callbacks: Adopted ADR 0006 for stable dev domains via Cloudflare.
    • Implement Authorization Code Grant flow.
    • “Connect eBay” UI and backend callback handler.
    • Prompt Hardening: Implemented XML-style structural delimiters in config.py.
    • Injection Mitigations: Added negative constraints and system-level security instructions to all AI prompts.
    • Security Analysis: Published analysis-llm-security.md detailing PI and resource exhaustion risks.
    • Fetch and store user’s eBay Shipping, Payment, and Return policies.
    • Implement “Push to eBay” via the Inventory/Offer API.

Phase 5: Cloud Native & Deployment (Completed)

    • Add Caddy reverse proxy to mise orchestration.
    • Explicitly configured non-privileged ports (8443/3443).
    • Network Consolidation: Unified API access via /api on port 3443 and added stable ingress port 8080.
    • Backend: Multi-stage Dockerfile using uv with non-root user mapping.
    • Frontend: Multi-stage Dockerfile utilizing Next.js standalone optimization.
    • Orchestration: Root docker-compose.yaml for full-stack lifecycle management.
    • Integrated QEMU MicroVM isolate task with AppArmor and Virtio-9p.
    • Unified secret injection via fnox and mise across host, docker, and VM.
