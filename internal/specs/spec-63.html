SPEC-63: Ephemeral Development Sandboxes

1. Goal

Introduce an “Ephemeral Sandbox” capability that allows developers to spin up a completely isolated, pre-seeded instance of the entire stack in temporary directories or Docker volumes.

2. Traceability

  • Phase: Phase 8: Production Readiness (DX/QA).
  • Objective: Protect the primary development state during destructive testing or complex bug reproduction.

3. Implementation Strategy

  1. Sandbox Script: Create scripts/spawn-sandbox.sh that creates a temporary workspace.
  2. Pre-Seeded Data: Automatically populate the sandbox database with a known “Golden State” (from SPEC-60 snapshots).
  3. Isolation: Ensure the sandbox instance uses its own isolated ports and log files to prevent interference with the main development environment.
