SPEC-62: Environment Feature Toggles

1. Goal

Replace the fragmented and manual management of multiple mise.*.toml profiles with a single, environment-variable-driven configuration. This simplifies the development “Mental Model” and reduces configuration drift.

2. Traceability

  • Phase: Phase 8: Production Readiness (DX).
  • Objective: Eliminate the manual burden of switching between hybrid, live, local, and other environment profiles.

3. Implementation Strategy

  1. Unified Configuration: Centralize environment variable management in a single mise.local.toml (ignored by git).
  2. Feature Flags: Use a primary variable like THREADSCOUT_MODE (e.g., mock, local, production) to toggle behaviors like eBay mocks or OAuth providers.
  3. Cleanup: Deprecate and remove redundant mise.hybrid.toml, mise.tailnet.toml, etc.
