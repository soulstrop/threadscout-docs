SPEC-61: Unified Root Orchestration

1. Goal

Consolidate all core backend, frontend, and infrastructure tasks into a single root-level mise.toml. This allows developers to manage the entire application stack from the project root using unified commands.

2. Traceability

  • Phase: Phase 8: Production Readiness (DX).
  • Objective: Simplify onboarding and CI/CD by providing a single point of entry for all project operations.

3. Implementation Strategy

  1. Orchestration Tasks: Move dev, test, lint, and build tasks to the root mise.toml.
  2. Parallel Execution: Leverage mise task dependencies or concurrently (if needed) to start both backend and frontend services with one command (mise run dev).
  3. Path Normalization: Ensure all root-level tasks correctly reference the backend/ and frontend/ subdirectories.
