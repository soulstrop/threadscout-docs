SPEC-69: Modular Application Bootstrapping

1. Goal

Refactor the monolithic main.py by offloading middleware, exception handling, and lifecycle events to dedicated modules. This turns main.py into a clean “manifest” of the application’s routes and global configuration.

2. Traceability

  • Phase: Phase 8: Production Readiness (DX).
  • Objective: Improve the readability of the application’s entry point and simplify the addition of new global middlewares or handlers.

3. Target Files

  • backend/src/threadscout/bootstrap.py: (New) Initialization logic for Sentry, Prometheus, and Middleware.
  • backend/src/threadscout/errors.py: (New) Global exception handlers and custom error types.
  • backend/src/threadscout/main.py: Clean entry point import routes and bootstrap.setup_app(app).

4. Implementation Strategy

  1. Extract Lifecycle: Move @app.on_event("startup") and shutdown tasks to bootstrap.py.
  2. Modular Middleware: Create helper functions in bootstrap.py for configuring CORS, Sentry, and custom request logging.
  3. Exception Mapping: Consolidate HTTPException handlers into errors.py.
