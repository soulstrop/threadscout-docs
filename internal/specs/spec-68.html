SPEC-68: Streamlined eBay Service Layer

1. Goal

Consolidate the fragmented eBay service files into a single, well-structured package. This reduces the “file-surfing” required to manage the project’s most complex integration.

2. Traceability

  • Phase: Phase 8: Production Readiness (DX).
  • Objective: Simplify the implementation of new eBay API features (e.g., Video uploads, multi-quantity listings).
  • User Journey: [UJ-05] Draft Synthesis & Push.

3. Target Files

  • backend/src/threadscout/services/ebay/: (New Package)
  • backend/src/threadscout/services/ebay/client.py: The core API wrapper (from ebay_impl.py).
  • backend/src/threadscout/services/ebay/mock.py: The developer mock (from ebay_mock.py).
  • backend/src/threadscout/services/ebay/schemas.py: eBay-specific Pydantic models.

4. Implementation Strategy

  1. Refactor Factory: Update the ebay_service_factory to look for implementations within the new package.
  2. Schema Isolation: Move large eBay-specific JSON schemas (Categories, Policies) into a sub-directory to keep the logic files clean.
  3. Consistent Interface: Ensure that MockEbayService and EbayService strictly adhere to the same Abstract Base Class (ABC).
