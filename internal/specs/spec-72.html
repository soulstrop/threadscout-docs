Spec 72

SPEC-72: Automated Image Processing & C2PA Provenance Pipeline

1. Goal

Design an automated image processing pipeline for ingested product photographs. The pipeline will optimize images for cost and speed, automatically correct orientation, remove distracting backgrounds, label the image, and cryptographically certify every structural change using the C2PA standard defined in SPEC-71.

This spec enables Journey [UJ-01] Item Ingest (Mobile) and [UJ-02] AI Extraction.

2. Background

Raw photos ingested from mobile devices are often large (leading to high LLM token costs and latency), unoriented, and contain cluttered backgrounds that degrade the aesthetic of the final eBay draft. Furthermore, multiple photos in an item group need to be semantically distinguished (e.g., “Front”, “Tag”, “Flaw”) for automated listing generation.

3. Implementation Strategy

The pipeline will be executed asynchronously when a batch of images is uploaded to storage/{user_id}/{item_id}/.

Pipeline Steps

  1. Optimization (Resize):
    • Load raw image using Pillow.
    • Downscale the image so its longest edge is at a maximum threshold (e.g., 1024px or 1536px), maintaining aspect ratio. This is the optimal resolution for Gemini Vision token usage without losing legible detail.
    • C2PA Action: c2pa.resized
  2. Rotation (Right-Side Up):
    • First pass: Read EXIF orientation tags and apply correction.
    • Second pass (Optional/AI): If no EXIF exists or if the photo is still suspected to be sideways (common with overhead flat-lays), use a lightweight vision model to predict the correct “up” vector and rotate.
    • C2PA Action: c2pa.orientation.changed
  3. Background Masking & Replacement:
    • Use a dedicated background removal model (e.g., rembg with U-Net architecture) or an equivalent LLM/Vision tool to segment the primary clothing item.
    • Create a composite image where the masked subject is placed onto a neutral, solid background (e.g., solid white #FFFFFF or a light studio gray).
    • C2PA Action: c2pa.removed.background (or c2pa.edited)
  4. Keyword Labeling (AI Tagging):
    • Pass the processed image to the Gemini Vision model (via services/ai.py) to generate contextual labels.
    • Prompt the LLM to classify the photo relative to the item group (e.g., "front_full", "back_full", "label_detail", "fabric_texture", "flaw_stain").
    • These labels will be saved as metadata alongside the image in the database.
  5. C2PA Certification:
    • Compile the list of actions performed (c2pa.resized, c2pa.orientation.changed, c2pa.removed.background).
    • Invoke sign_image_with_c2pa() (from SPEC-71) to embed the final cryptographic manifest into the optimized .jpg or .png.

7. Error Handling & Failsafes

  • Non-Destructive Processing: The original raw ingested file must always be preserved in storage/{user_id}/{item_id}/raw/. Processed images should be saved in an optimized/ or processed/ subdirectory.
  • Background Removal Failure: If rembg or the AI masking tool fails or times out, the pipeline should gracefully fall back to the original (resized) background, logging the failure, and omitting the c2pa.removed.background action.
  • AI Rate Limits: If the keyword labeling call hits LLM rate limits, the image should still be saved and signed. The system should mark the image’s labels as ["pending"] for a retry queue or default to ["unclassified"].
  • Async Processing: Given the CPU-intensive nature of background removal and the latency of LLM calls, this pipeline must be run asynchronously (e.g., via FastAPI BackgroundTasks or a dedicated queue like Celery/RQ in the future) to avoid blocking the client upload.
