SPEC-70: Programmatic Session Injection for E2E Tests

Objective

Implement a secure, headless authentication bypass mechanism for Playwright to allow end-to-end (E2E) tests to authenticate in hardened environments (e.g., hybrid, production) where the “Developer Login (Mock)” UI is intentionally disabled.

Background

Currently, Playwright relies on clicking the “Developer Login (Mock)” button in the UI. In hardened environments, this button is removed to prevent security risks. Consequently, E2E tests fail during the auth.setup.ts phase. To test the fully authenticated application behind a reverse proxy (e.g., hybrid profile), Playwright needs a reliable way to inject a valid NextAuth session.

Requirements

  1. Test-Only Authentication Endpoint:
    • Create a specialized Next.js API route (e.g., app/api/auth/e2e-inject/route.ts).
    • This endpoint must ONLY allow access if a specific, highly secure secret (e.g., THREADSCOUT_BYPASS_BETA_TOKEN or a dedicated E2E_INJECTION_SECRET) is provided in the Authorization header.
    • The endpoint should generate a valid NextAuth session token (JWT) for a mock user.
  2. Playwright Context Integration:
    • Update frontend/tests/auth.setup.ts.
    • Instead of visiting /login and clicking the mock button, Playwright should make a server-to-server HTTP POST request to the injection endpoint.
    • Extract the next-auth.session-token (or __Secure-next-auth.session-token) cookie from the response.
    • Inject this cookie directly into the Playwright browser context using page.context().addCookies([...]).
  3. Security Measures:
    • Ensure the injection endpoint explicitly checks process.env.NODE_ENV !== 'production' OR strictly validates the shared secret to prevent malicious session generation in real production deployments.
    • Preferably, disable the route entirely in true production builds if possible, relying purely on the secret for hybrid/staging checks.

Implementation Steps

  1. Backend/Next.js Auth Route:
    • Analyze the current NextAuth configuration to understand how the JWT is encoded (usually using the NEXTAUTH_SECRET).
    • Write a utility function or API route that signs a mock user payload using the same NextAuth JWT logic.
  2. Playwright Refactor:
    • Refactor auth.setup.ts to implement the new fetch -> inject cookie -> navigate sequence.
    • Ensure auth.setup.ts falls back to the new method transparently across dev and hybrid profiles.
  3. Validation:
    • Run FNOX_PROFILE=dev mise run check:e2e to verify standard operation.
    • Run FNOX_PROFILE=hybrid mise run check:e2e to verify tests pass without the UI mock button.

Success Criteria

The Playwright E2E suite successfully logs in and runs all authenticated workflow tests in the hybrid profile without relying on any visible mock provider buttons on the login page.
