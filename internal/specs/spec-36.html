Technical Specification: SPEC-36

Status: READY

Title: Production Auth Hardening & Secret Enforcement

Journey Traceability

  • This spec hardens the security posture of the [UJ-00] SaaS Authentication and [UJ-08] Operations & Triage journeys.
  • It addresses findings from the Pass 5 Security Audit.

1. Goal

Eliminate “predictable default” vulnerabilities in authentication secrets and enforce strict validation for production-grade environments (TUNNEL mode). This ensures that ThreadScout cannot be deployed to a public domain with development-only or well-known secrets.

2. Target Files

Backend (Python/FastAPI)

  • backend/src/threadscout/config.py (Update):
    • Change ops_token default to None or a high-entropy random placeholder.
    • Update validate_production_security to require a non-default ops_token.

Frontend (Next.js/TypeScript)

  • frontend/src/lib/auth_profiles.ts (Update):
    • Implement a “Fail Securely” check in the TUNNEL case of getAuthProfile.
    • Throw a critical error if process.env.THREADSCOUT_NEXTAUTH_SECRET is missing or matches the well-known “default-secret-for-development-only” string.

3. Implementation Strategy

3.1 Backend: Ops Token Enforcement

The Settings class must treat the ops_token as a sensitive credential.

  • Development: Fallback to a “dev-only-token” is permitted in DIRECT mode.
  • Production: If app_env == "production" or ingress_mode == TUNNEL, the validator must raise a ValueError if the token is missing or insecure.

3.2 Frontend: Secret Nonce Integrity

The getAuthProfile factory will be updated to include a security gate:

if (ingressMode === IngressMode.TUNNEL) {
  const secret = process.env.THREADSCOUT_NEXTAUTH_SECRET;
  if (!secret || secret === "default-secret-for-development-only") {
    throw new Error("SECURITY VIOLATION: A strong THREADSCOUT_NEXTAUTH_SECRET is required for public tunnel deployments.");
  }
}

4. Verification Plan

4.1 Negative Test (Backend)

  1. Set THREADSCOUT_APP_ENV=production.
  2. Unset THREADSCOUT_OPS_TOKEN.
  3. Success Criteria: The backend fails to start with a clear validation error.

4.2 Negative Test (Frontend)

  1. Set THREADSCOUT_INGRESS_MODE=TUNNEL.
  2. Set THREADSCOUT_NEXTAUTH_SECRET=default-secret-for-development-only.
  3. Success Criteria: The application fails to build or initialize, throwing the security violation error.
