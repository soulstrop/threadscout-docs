SPEC-67: Unified Auth Architecture

1. Goal

Consolidate fragmented authentication logic spread across auth.py, auth_strategies.py, and security.py. This provides a single, cohesive source of truth for identity management, token verification, and password hashing.

2. Traceability

  • Phase: Phase 8: Production Readiness (DX).
  • Objective: Lower the barrier to adding new auth providers (e.g., Apple, Microsoft) and simplify auth debugging.
  • User Journey: [UJ-00] SaaS Authentication.

3. Target Files

  • backend/src/threadscout/auth/: (New Package)
  • backend/src/threadscout/auth/core.py: Token verification, hashing (from security.py).
  • backend/src/threadscout/auth/strategies/: OAuth implementations (from auth_strategies.py).
  • backend/src/threadscout/auth/dependencies.py: FastAPI Depends (from auth.py).

4. Implementation Strategy

  1. Unify Security: Move Fernet and hashing utilities into a clean auth/crypto.py or auth/core.py.
  2. Strategy Pattern: Formalize the OAuth strategy pattern to make it strictly interface-driven.
  3. Dependency Cleanup: Ensure the get_current_user logic is consolidated and doesn’t rely on global state outside of the Settings.
