ADR 0007: Modular Reverse Proxy Configuration via Caddy Snippets

Status

Proposed

Context

ThreadScout utilizes Caddy as a unified ingress point for both local development and public tunnel access. As the application grows (adding identity providers, static assets, and SaaS terminal states), the Caddyfile has become repetitive. Currently, the logic for routing API calls, Auth.js requests, and frontend pages is duplicated across the localhost:3443 block and the public tunnel block.

This duplication introduces:

  1. Configuration Drift: Changes to API routing must be manually synchronized in two places.
  2. Hardcoded Specificity: Domain names (e.g., threadscout.constans.dev) are hardcoded, reducing portability.
  3. Maintenance Overhead: Increasing complexity makes the file harder to audit for security.

Decision

We will refactor the Caddyfile to use Caddy Snippets. All core routing logic will be defined once in a reusable snippet, which is then “imported” into specific site blocks representing different execution environments.

Additionally, we will parameterize the public hostname using an environment variable (THREADSCOUT_PUBLIC_HOSTNAME) to improve portability.

Options Considered

1. The Monolithic Block (Current)

  • Pros: Simple visibility.
  • Cons: High risk of drift; redundant code.

2. Multiple File Strategy (Caddyfile.dev, Caddyfile.prod)

  • Pros: Strict physical isolation.
  • Cons: High maintenance; environment parity is difficult to guarantee.

3. Modular Snippets (Selected)

  • Pros: DRY (Don’t Repeat Yourself); guarantees environment parity; allows for environment-specific extensions (e.g., adding production-only security headers).
  • Cons: Requires understanding of Caddy’s import syntax.

Architectural Visualization

Consequences

  • Environment Parity: The logic used during local development is identical to the logic used in the public tunnel, reducing “deployment surprises.”
  • Portability: The Caddyfile is now domain-agnostic. New developers can stand up the stack by simply providing a THREADSCOUT_PUBLIC_HOSTNAME env var.
  • Cleanliness: The site blocks are reduced to a few lines each, clearly showing where the traffic comes from, while the snippet defines what happens to it.
