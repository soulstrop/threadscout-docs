ADR 0009: Hybrid Hosting Strategy (GitHub Pages + Dynamic Sub-domain)

Status

Proposed

Context

ThreadScout is a full-stack application requiring a server-side runtime for AI processing, eBay API integration, and Auth.js session management. However, public-facing components like the marketing landing page, documentation, and waitlist do not strictly require a dynamic backend for every request.

Currently, all traffic is routed through a Cloudflare Tunnel to a local/private instance. This exposes the private infrastructure to unnecessary crawler traffic and creates a single point of failure for the public “face” of the product.

Decision

We will adopt a Hybrid Hosting Strategy:

  1. Static Layer (GitHub Pages):
    • Domain: threadscout.constans.dev
    • Content: Marketing Landing Page, Public Documentation (/docs), and the Waitlist Signup form.
    • Technology: Next.js Static Export (output: 'export') and Pandoc-generated HTML.
  2. Dynamic Layer (Cloudflare Tunnel):
    • Domain: app.threadscout.constans.dev
    • Content: Reseller Dashboard, Ingest Gallery, Synthesis Editor, and FastAPI Backend.
    • Technology: Docker-orchestrated Next.js (Node.js) and FastAPI.

Architectural Visualization

Consequences

  • Resilience: The marketing site and documentation remain online even if the local development instance is powered down.
  • Performance: Static assets are served from GitHub’s global CDN, improving SEO and LCP for first-time visitors.
  • Security: Reduces the attack surface of the private infrastructure by offloading public-facing pages to a static environment.
  • Complexity: Requires managing two deployment targets and configuring cross-subdomain CORS for the Waitlist API.
  • Portability: Aligns with modern SaaS patterns where “Marketing” and “Product” are decoupled.
