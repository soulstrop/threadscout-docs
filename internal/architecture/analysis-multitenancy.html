Engineering Analysis: Multitenancy Migration

Date Created: February 15, 2026
Last Updated: February 28, 2026
Status: Implementation Verified (Phase 7/8)

Executive Summary

This document tracks the evolution of ThreadScout from a single-tenant local utility to a secure, multi-tenant SaaS-ready platform. As of February 28, 2026, the core logical and physical isolation layers have been implemented and verified.

2. Implementation Status (Verified Feb 28, 2026)

A. Identity Foundation

  • OIDC Integration: Implemented PocketID as a self-hosted OIDC sidecar.
  • Brokerage: Auth.js (NextAuth) serves as the frontend broker for multiple providers (Google, Apple, eBay).
  • Just-In-Time (JIT) Provisioning: Users are automatically created in the database upon their first successful OIDC login.

B. Logical Data Isolation

  • User-Scoped Queries: All database models (Item, Image, EBayToken, EBayPolicy) now include a user_id field.
  • Dependency Injection: The get_current_user dependency in FastAPI enforces isolation at the API level, ensuring users can only access their own records.
  • Security Middleware: Cryptographic JWT signature verification is active for every cross-origin request.

C. Physical Asset Isolation

  • Namespaced Storage: The local filesystem has been restructured to storage/{user_id}/{item_id}/.
  • Migration: A migration script was successfully executed to move legacy storage/ contents into their respective user silos.
  • S3 Abstraction (BYOS): (In Progress) A pluggable StorageProvider spec has been finalized to allow for S3-compatible cloud storage and per-user Bring Your Own Storage (BYOS).

3. SaaS Logic & Quotas

  • Tiers & Quotas: Implemented User.tier (Free/Pro) and monthly limits for AI tokens and listings.
  • Billing Integration: (Phase 8 - Planned) Integration with Stripe to link subscription states to user tiers.
