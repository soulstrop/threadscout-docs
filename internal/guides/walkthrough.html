1. Configuration and Environment

ThreadScout uses mise for toolchain management and Pydantic Settings for application configuration. This ensures consistency across development, mocking, and production environments.

2. Authentication Strategy

ThreadScout supports two modes: Zero-Config Local Mode and Secure SaaS Mode (OIDC).

In local mode (enable_auth=False), the backend automatically provisions a local-user. In SaaS mode, it verifies JWTs from trusted OIDC providers like PocketID or Google.

The get_current_user dependency in backend/src/threadscout/auth.py handles this logic:

4. AI-Powered Analysis and Enrichment

Once images are ingested, the user triggers the creation of a new item via the /api/create-item endpoint in backend/src/threadscout/api/items.py. This endpoint orchestrates the AI Vision analysis and enrichment process:

  1. AI Vision: Google Gemini 2.0 Flash extracts brand, size, material, and condition from the photos.
  2. Enrichment: (Optional) Scrapes a provided URL and uses Gemini 2.5 Flash-Lite to validate and refine the metadata.

The create_item endpoint handles this workflow:

sed -n '40,100p' backend/src/threadscout/api/items.py

logger = structlog.get_logger(__name__)

router = APIRouter(prefix="/api", tags=["items"])


def get_ingest_provider(
    current_user: User = Depends(get_current_user),
    session: Session = Depends(get_session),
) -> IngestProvider:
    """Dependency to provide the configured ingest backend for the user."""
    db_backend = session.exec(
        select(UserSetting).where(
            UserSetting.user_id == current_user.id, UserSetting.key == "ingest_backend"
        )
    ).first()
    db_config = session.exec(
        select(UserSetting).where(
            UserSetting.user_id == current_user.id, UserSetting.key == "ingest_config"
        )
    ).first()

    ingest_backend = db_backend.value if db_backend else settings.ingest_backend
    ingest_config = db_config.value if db_config else settings.ingest_config

    if ingest_backend == "google_photos" and ingest_config:
        return GooglePhotosProvider(ingest_config)

    # Default to Local
    base_ingest_path = (settings.base_dir / settings.ingest_dir_name).resolve()

    if ingest_backend == "local" and ingest_config:
        # Security: Prevent absolute paths or escaping the base ingest directory
        # We treat ingest_config as a relative subpath
        p = Path(ingest_config).expanduser()

        # If it's absolute, we only allow it if it's already within base_ingest_path
        # Otherwise, we force it to be relative to base_ingest_path
        if p.is_absolute():
            resolved_p = p.resolve()
        else:
            resolved_p = (base_ingest_path / p).resolve()

        if not resolved_p.is_relative_to(base_ingest_path):
            logger.warning("User attempted to escape ingest boundary", user_id=current_user.id, path=ingest_config)
            ingest_path = base_ingest_path
        else:
            ingest_path = resolved_p
    else:
        ingest_path = settings.ingest_dir_path or base_ingest_path

    return LocalIngestProvider(ingest_path)


def get_storage_provider(
    current_user: User = Depends(get_current_user),
) -> StorageProvider:

5. Storage and Media Management

After analysis, the /api/create-item endpoint moves files from the ingest area to a permanent, isolated storage location. ThreadScout uses a Storage Provider abstraction to support both local filesystems and S3-compatible buckets.

The create_item endpoint names the storage path using a slugified title and a short ID to ensure human-readability while maintaining uniqueness.

6. eBay Integration (OAuth and Policies)

ThreadScout securely integrates with eBay using the OAuth 2.0 Authorization Code Flow. Sensitive tokens are encrypted at rest using Fernet (AES-128).

The ebay router in backend/src/threadscout/api/ebay.py handles the initial redirection and the callback:

7. Publishing to eBay

The final step in the reseller loop is pushing the analyzed and enriched item to eBay as a draft offer. This involves:

  1. Condition Mapping: Converting natural language condition into eBay’s enum.
  2. Inventory Creation: Creating an inventory record with photos and aspects.
  3. Offer Creation: Defining the price, category, and fulfillment policies.

The push_item_to_ebay endpoint orchestrates this final push:
