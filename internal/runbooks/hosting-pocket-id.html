Hosting Pocket Id

Runbook: Self-Hosting PocketID (OIDC Provider)

See also: PocketID Identity Flow & Troubleshooting for sequence diagrams and environmental configurations.

This document outlines the deployment and configuration of PocketID for a passwordless, passkey-based identity solution on Ubuntu 24.04.

I. Prerequisites Checklist

Ensure the following are available on your workstation:

  • Docker & Docker Compose: Installed and active (managed via mise).
  • Reverse Proxy: Caddy, Nginx, or Traefik configured for SSL.
  • Domain Name: A subdomain (e.g., auth.example.com) pointing to your host.
  • HTTPS: Mandatory for WebAuthn/Passkey functionality.

II. Environment Preparation

Run the following commands in your fish shell to create the directory structure and environment file.

# Create application directory
mkdir -p ~/services/pocketid/data
cd ~/services/pocketid

# Create environment configuration
cat <<EOF > .env
APP_URL=[https://auth.yourdomain.com](https://auth.yourdomain.com)
PUID=$(id -u)
PGID=$(id -g)
TRUST_PROXY=true
EOF

III. ThreadScout Integration

To ensure seamless OIDC discovery and token verification regardless of network topology, ThreadScout uses unified environment variables for PocketID orchestration.

In your environment configuration (e.g., mise.toml or docker-compose.yaml), you must define:

  1. THREADSCOUT_POCKET_ID_PUBLIC_URL: The browser-accessible URL for authentication flows (e.g., https://auth.threadscout.com or https://auth.localhost:3443).
  2. THREADSCOUT_POCKET_ID_INTERNAL_URL: The backend-to-backend URL used for fetching the JWKS (JSON Web Key Set) without routing through the public ingress (e.g., http://pocket-id:1411 or http://localhost:8081).
  3. THREADSCOUT_POCKET_ID_CLIENT_ID: The client identifier registered in PocketID (default: threadscout).
  4. THREADSCOUT_POCKET_ID_CLIENT_SECRET: The secret used for token exchange, securely managed via fnox.

ThreadScout’s configuration layer will automatically prioritize the THREADSCOUT_POCKET_ID_INTERNAL_URL for backend verifications while using the THREADSCOUT_POCKET_ID_PUBLIC_URL for frontend NextAuth redirects.

IV. First Run Initialization

After the container is started for the first time, you must perform the following administrative tasks:

  1. Admin Setup: Navigate to your THREADSCOUT_POCKET_ID_PUBLIC_URL and follow the prompts to create the initial administrative account.
  2. Client Registration: Go to the Clients section (e.g., /admin/clients) and create a new client with ID threadscout.
  3. Client Secret: Copy the generated Client Secret and store it in fnox under the key THREADSCOUT_POCKET_ID_CLIENT_SECRET.
  4. Redirect URIs: Add the following Callback URLs to the threadscout client configuration:
    • https://threadscout.constans.dev/api/auth/callback/pocketid
    • https://localhost:3443/api/auth/callback/pocketid
  5. Scopes: Ensure the client has openid, profile, and email scopes enabled (these are typically default).
